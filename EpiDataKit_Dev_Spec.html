EpiDataKit — Dev Spec

疾病流行病学数据聚合平台 · 两个独立 Demo 的完整技术规格
Phase: Demo MVP
Timeline: 8 weeks
Demos: BioEpi + TobaccoEpi
Shared infra: Yes (separate brand)
Demo A · BioEpi
生物医药 & 医疗器械
  • 客户：Biotech 市场准入团队
  • 场景：NICE STA / EU JCA / CADTH 档案
  • 输入：疾病名称 / ICD-10
  • 输出：HTA 格式流行病学章节草稿
  • 品牌色：蓝色系 #1B4F72
Demo B · TobaccoEpi
烟草 & 尼古丁产品（独立品牌）
  • 客户：电子烟 / 尼古丁袋公司监管团队
  • 场景：FDA PMTA / MRTP 申请
  • 输入：市场 + 产品类型 + 监管路径
  • 输出：吸烟率 + 归因疾病负担 + 切换模拟
  • 品牌色：琥珀色系 #784212
⚠️ 品牌隔离强制要求：两个 Demo 不得共用 Logo、主色、域名。前端路由 /bioepi/* 与 /tobaccoepi/* 完全独立，页面之间无跳转链接。TobaccoEpi 每页必须展示法律免责声明 Banner。

BioEpi — 数据管道实现

Step 1 · ICD-10 映射

# 优先本地词库 fuzzy match，fallback 到 WHO ICD API

# 1a. 本地匹配（推荐：预下载 ICD-10 CM CSV）
icd_codes = icd_search(query)  # 返回 ['E11', 'E11.9', ...]

# 1b. Fallback: WHO ICD-11 API
GET https://id.who.int/icd/release/11/2024-01/mms/search?q={query}

# 1c. GBD cause_id 映射（预建静态映射表）
cause_ids = icd_to_gbd_cause(icd_codes)  # e.g. [587] for type 2 diabetes

Step 2 · 并发数据拉取

async def fetch_bioepi_data(cause_ids, markets, year_range):
    tasks = [
        fetch_gbd_csv(cause_ids, markets, year_range),   # 本地缓存的 CSV
        fetch_who_gho(icd_codes, markets),                # OData API
        fetch_eurostat(icd_codes, markets),               # REST API
        fetch_seer(icd_codes) if "US" in markets else None,
    ]
    results = await asyncio.gather(*[t for t in tasks if t])
    return merge_and_normalize(results)

Step 3 · 标准化数据结构

# 所有数据源统一输出此结构，前端只需处理一种格式
NormalizedRecord = {
    "disease":    str,    # "type 2 diabetes"
    "icd_code":   str,    # "E11"
    "metric":     str,    # prevalence | incidence | daly_rate | mortality
    "value":      float,
    "unit":       str,    # per_100k | absolute | percentage
    "country":    str,    # ISO3: "USA", "GBR", "DEU"
    "year":       int,
    "age_group":  str,    # "all_ages" | "15-49" | "50+"
    "sex":        str,    # "both" | "male" | "female"
    "source":     str,    # "GBD2023" | "WHO_GHO" | "Eurostat" | "SEER"
    "ui_lower":   float,  # 95% UI，GBD 专有，其他来源可为 null
    "ui_upper":   float,
    "citation":   str,    # 已格式化的引用字符串

TobaccoEpi — 数据管道

模块 A · 吸烟率数据（WHO GHO）

# 核心 WHO GHO 烟草指标代码
M_Est_smk_curr          # 当前吸烟率（按性别）
TOBACCO_0000000192      # 15岁以上年龄标准化吸烟率
TOBSMOKEDAILY           # 每日吸烟率
TOBACCO_0000000194      # 青少年（13-15岁）吸烟率
TOBCESSATION            # 戒烟服务可及性

# API 调用示例
GET https://ghoapi.azureedge.net/api/M_Est_smk_curr
    ?$filter=SpatialDim eq 'USA' and TimeDim ge 2015 and TimeDim le 2024
    &$select=SpatialDim,TimeDim,Dim1,NumericValue,Low,High

# 返回字段：
# Dim1         = SEX: MLE(男) / FMLE(女) / BTSX(合计)
# NumericValue = 吸烟率 %
# Low / High   = 95% CI

模块 B · 烟草归因疾病负担（PAF 计算）

# GBD 2023 烟草风险因子 REI ID = 99 (smoking)
# 关注疾病（与 PMTA/MRTP 核心关切对应）：
TOBACCO_ATTRIBUTED_CAUSES = {
    "C33-C34": "lung_cancer",        # cause_id = 426
    "J44":     "COPD",               # cause_id = 509
    "I20-I25": "ischemic_heart",    # cause_id = 493
    "I60-I69": "stroke",             # cause_id = 494
    "C15":     "esophageal_cancer", # cause_id = 411
}

# PAF（人群归因分数）计算
def calc_paf(smoking_dalys: float, total_dalys: float) -> float:
    return smoking_dalys / total_dalys if total_dalys > 0 else 0

# GBD CSV 字段：location / year / cause / sex / age / measure / val / upper / lower
# measure = "DALYs" 或 "Deaths"，按 rei="smoking" 筛选

TobaccoEpi — 切换收益模拟器

⚠️ relative_harm 不由本工具提供。此参数必须由客户基于自有毒理学/生物标志物研究填写，并在 UI 中以红色边框 + 警告文字标注。工具只提供计算框架。 
# 核心公式（简化版，Demo 阶段）
# 避免的 DALYs = 吸烟人数 × 切换率 × PAF × (1 - 相对危害系数) × 年份权重

def switching_impact(
    smoker_population: int,    # 目标市场当前吸烟人数（来自模块A）
    switch_rate:       float,  # 切换率 0.0–1.0，用户拖滑块输入
    paf_dict:          dict,   # 各疾病 PAF（来自模块B）
    relative_harm:     float,  # ⚠️ 客户自填：0.0=无害，1.0=等同香烟
    horizon_years:     int = 10
) -> dict:
    avoided = {}
    for cause, paf in paf_dict.items():
        annual = smoker_population * switch_rate * paf * (1 - relative_harm)
        avoided[cause] = {
            "annual_avoided_dalys":      round(annual),
            "cumulative_10yr_dalys":     round(annual * horizon_years),
            "note": "Simplified model. Not for regulatory submission."
        }
    return avoided
